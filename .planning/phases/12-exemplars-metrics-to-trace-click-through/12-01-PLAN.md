<objective>
Activate the SDK layer of the three-layer exemplar plumbing:

1. Add `.setExemplarFilter(ExemplarFilter.traceBased())` to both services' `buildMeterProvider()` methods (EXMP-01 / D-E11).
2. Restructure `HttpServerSpanFilter.doFilterInternal()` from try-with-resources to manual scope management so `Span.current()` is valid when the histogram fires (D-E1 / F3-1 mitigation).

Purpose: Without ExemplarFilter set, the SDK never attaches trace_id/span_id to histogram data points. Without the scope fix, ExemplarFilter sees an invalid span context and attaches no exemplar even when the filter IS set.

Output: Both OtelSdkConfiguration.java files gain one import + one builder call. HttpServerSpanFilter.java changes one try-with-resources to manual try-finally with scope.close() last.
</objective>
