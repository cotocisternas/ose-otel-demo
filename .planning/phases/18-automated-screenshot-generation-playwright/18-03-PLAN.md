Step A: Verify prerequisites before running.

Run these checks and abort if any fails:
```bash
# 1. Syntax check on the capture script
node --check scripts/capture-screenshots.mjs

# 2. Verify infra is up (Grafana must be reachable)
curl -sf http://localhost:3000/api/health || echo "ERROR: Grafana not reachable — run: mise run infra:up"

# 3. Verify producer service is up (needed for v2.0 HEAD captures)
curl -sf http://localhost:8080/actuator/health || echo "WARN: Producer service not reachable — run: mise run dev"
```

Step B: Install Playwright in the scripts/ directory.
```bash
cd scripts
PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1 npm install
cd ..
```
Note: PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1 prevents downloading Playwright's bundled Chromium — the script uses system Chromium at /usr/bin/chromium (auto-detected in capture-screenshots.mjs). If system chromium is not present, run `cd scripts && npx playwright install chromium` instead.

Step C: Run FORCE=1 to produce all v2.0 new captures (step-04-metrics, step-11-OFF, step-11-ON, step-12-exemplars). v1.0 PNGs already exist and will be skipped by default.
```bash
FORCE=1 mise run screenshots
```

The command will:
1. Validate Grafana at localhost:3000/api/health
2. Start scripts/load.sh background child (SLOW_RPS=2)
3. Wait 30s warm-up
4. Skip v1.0 tag-cycling captures (PNGs already exist; FORCE=1 recaptures them too — expect 30-60 min for full run with tag cycling)

ALTERNATIVELY if only the 4 new v2.0 captures are needed (faster):
```bash
mise run screenshots
```
Without FORCE=1, the script skips all existing v1.0 PNGs (step-01 through step-06) since they exist on disk. Only the 4 new v2.0 PNGs (step-04-metrics, step-11-OFF, step-11-ON, step-12-exemplars) will be captured since they do not exist yet.

Step D: Verify all 4 new PNGs were created.


All four files must exist and have non-zero size (> 10KB indicates actual content, not a blank frame).
