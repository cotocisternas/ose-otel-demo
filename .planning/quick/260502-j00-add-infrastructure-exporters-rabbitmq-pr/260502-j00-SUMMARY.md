## Resume work — bugs fixed and panels added

Live validation of the dashboard panels through the Grafana MCP server (anonymous-Admin, no token needed) surfaced two bugs left in the j00 plan's must_haves that a static-name check would have missed:

**Bug 1: aggregate-only `rabbitmq_queue_messages`.** The `rabbitmq_prometheus` plugin defaults to `/metrics`, which emits aggregate-only series with no `{queue=...}` label. The dashboard's `queue="orders.queue"` selectors silently returned empty. **Fix:** added `metrics_path: /metrics/per-object` to the rabbitmq scrape job in `prometheus.yaml`. After lgtm restart, `sum by (queue) (rabbitmq_queue_messages{queue!=""})` returns `{queue="orders.created"} 0` as expected.

**Bug 2: wrong queue name.** The actual queue is `orders.created`, not `orders.queue`. **Fix:** dropped the literal queue selector throughout — every panel that previously hard-coded a queue name now uses `sum by (queue) (...{queue!=""})` so the dashboard tracks whatever queues are declared.

**Enrichment ("better information from each exporter"):** rebuilt the dashboard from 18 → 22 panels, adding workshop-quality telemetry:

- **HTTP/AMQP row:** added `histogram_quantile(0.95, ...)` p95-latency-by-route alongside the RED rate panel.
- **Cache row:** added `rate(redis_commands_processed_total[1m])` (ops/sec) and folded `redis_connected_clients` + `rate(redis_evicted_keys_total[1m])` into a combined Connected-clients-and-evictions panel.
- **Database row:** added a Transactions-and-deadlocks panel with `xact_commit`/`xact_rollback`/`deadlocks` rates; expanded the Tuple-activity panel to also plot `tup_updated` and `tup_fetched`.
- **Broker row:** added an Acks-and-redeliveries-per-queue panel using `rabbitmq_queue_messages_acked_total` and `rabbitmq_queue_messages_redelivered_total`. The previous publish/deliver panel kept its global counters but the legend stopped saying "(received_total)".
